# LISTEN Time Clock 仕様書

放課後等デイサービス LISTEN 向け  
出勤・退勤打刻アプリ

---

# 1. 目的

本システムは、職員の **出勤時刻** および **退勤時刻** をタブレットから簡単に記録することを目的とする。

本システムは勤務時間計算や勤務表作成を主目的とせず、  
**タイムカードの代替として打刻ログを保存すること** を主目的とする。

あわせて、管理者が当日の全職員の出退勤状況を確認でき、必要に応じて打刻漏れ・直帰・修正忘れ等へ対応できるようにする。

---

# 2. システム概要

職員はタブレット画面から

- 自分の氏名を選択
- 出勤ボタン または 退勤ボタン を押す

ことで打刻を行う。

打刻データは Google スプレッドシートに保存する。  
従業員一覧も Google スプレッドシートから読み込む。

また、一覧画面では指定日の全職員の出退勤状況を表示する。  
管理者はパスワード入力後のみ、一覧画面から出勤時刻・退勤時刻を修正できる。

# 6. 画面仕様

## 6-1. 画面構成
本システムは以下の表示モードを持つ。

- 打刻
- 一覧

---

## 6-2. 打刻画面
打刻画面には以下を表示する。

- アプリ名
- 現在日時
- 従業員一覧
- 選択中の氏名
- 出勤ボタン
- 退勤ボタン
- 打刻メッセージ

### 操作手順
1. 従業員を選択
2. 出勤 または 退勤 ボタンを押す
3. 打刻ログを保存
4. 完了メッセージを表示
5. 一定時間後、選択状態をリセットする

---

## 6-3. 一覧画面
一覧画面では以下を表示する。

- 日付選択欄
- 表示ボタン
- 日別CSV出力ボタン
- 全職員の一覧表

一覧表の項目は以下とする。

| 項目 | 内容 |
|---|---|
| 氏名 | 職員名 |
| 出勤 | その日の最初の出勤時刻 |
| 退勤 | その日の最後の退勤時刻 |
| 状態 | 当日の勤務状態 |

### 初期表示
一覧画面を開いた時点では、**デフォルトで当日の日付** を表示する。

---

## 6-4. 管理者モード表示
管理者モード有効時のみ、一覧表の右端に **編集** を表示する。

例：

| 氏名 | 出勤 | 退勤 | 状態 |  |
|---|---|---|---|---|
| 田中 | 08:31 | 17:12 | 退勤済 | 編集 |
| 佐藤 | 09:02 |  | 勤務中 | 編集 |
| 鈴木 |  |  | 未出勤 | 編集 |

通常利用者には編集列を表示しない。

# 7. スプレッドシート構成

スプレッドシートには以下のシートを作成する。

---

## 7-1. employees
従業員名簿

| 列 | 項目 | 内容 |
|---|---|---|
| A | employee_id | 職員ID |
| B | name | 氏名 |
| C | active | 表示対象 |

### 例

| employee_id | name | active |
|---|---|---|
| E001 | 田中 | TRUE |
| E002 | 佐藤 | TRUE |
| E003 | 鈴木 | FALSE |

`active` が TRUE の従業員のみ表示する。

---

## 7-2. attendance_log
打刻ログ保存シート

**1打刻につき1行追加保存**

| 列 | 項目 |
|---|---|
| A | date |
| B | weekday |
| C | time |
| D | name |
| E | action |
| F | timestamp |

### 保存例

| date | weekday | time | name | action | timestamp |
|---|---|---|---|---|---|
| 2026/03/10 | 火 | 08:31:12 | 田中 | 出勤 | 2026/03/10 08:31:12 |
| 2026/03/10 | 火 | 17:12:01 | 田中 | 退勤 | 2026/03/10 17:12:01 |

### 重要事項
このシートは **元の打刻事実を残すためのログ** とする。  
管理者による修正が行われても、**このシートの値は変更・削除しない**。

---

## 7-3. attendance_edit
管理者修正データ保存シート

| 列 | 項目 | 内容 |
|---|---|---|
| A | date | 修正対象日 |
| B | name | 職員名 |
| C | clockIn | 修正後の出勤時刻 |
| D | clockOut | 修正後の退勤時刻 |
| E | editedAt | 修正日時 |
| F | editedBy | 修正者 |

### 例

| date | name | clockIn | clockOut | editedAt | editedBy |
|---|---|---|---|---|---|
| 2026/03/10 | 田中 | 08:31:12 | 18:00:00 | 2026/03/11 16:10:00 | 代表 |

### 重要事項
- 管理者修正はこのシートに保存する
- **元の打刻ログは残す**
- 一覧画面およびCSV出力では、修正データが存在する場合はそちらを優先する

---

# 8. 打刻ルール

## 8-1. 出勤打刻
氏名選択後、出勤ボタン押下で保存する。

## 8-2. 退勤打刻
氏名選択後、退勤ボタン押下で保存する。

## 8-3. 同日複数打刻
初版では、同一人物が同日に複数回打刻した場合でも、そのままログとして保存する。

## 8-4. 出勤なし退勤
初版では、出勤記録がない状態で退勤が押されても、そのままログとして保存する。

## 8-5. 打刻ミス
通常利用者は修正できない。  
必要に応じて管理者が管理者モードで修正する。

## 8-6. 連打防止
短時間に同一人物・同一種別の打刻が行われた場合は確認メッセージを表示する。

---

# 9. 一覧表示仕様

## 9-1. 一覧の表示対象
指定日または当日の `active=TRUE` の全職員を対象とする。

## 9-2. 集計ルール
通常ログから、職員ごとに以下を算出する。

- 出勤：その日の最初の「出勤」時刻
- 退勤：その日の最後の「退勤」時刻

修正データがある場合は、修正データを優先して表示する。

---

## 9-3. 状態判定ルール

| 状態 | 条件 |
|---|---|
| 未出勤 | 出勤・退勤ともに記録なし |
| 勤務中 | 出勤あり、退勤なし |
| 退勤済 | 出勤あり、退勤あり |
| 要確認 | 出勤なし、退勤あり |

※ 修正データがある場合は修正値を優先して判定する。

---

## 9-4. 一覧表示例

| 氏名 | 出勤 | 退勤 | 状態 |
|---|---|---|---|
| 田中 | 08:31 | 17:12 | 退勤済 |
| 佐藤 | 09:02 |  | 勤務中 |
| 鈴木 |  |  | 未出勤 |
| 山本 |  | 18:01 | 要確認 |

# 10. CSV出力仕様

## 10-1. 日別一覧CSV
一覧画面で表示中の日付を対象とし、全職員の一覧をCSV出力する。

### 出力項目
| 項目 | 内容 |
|---|---|
| 日付 | 対象日 |
| 氏名 | 職員名 |
| 出勤 | 出勤時刻 |
| 退勤 | 退勤時刻 |
| 状態 | 未出勤 / 勤務中 / 退勤済 / 要確認 |

---

## 10-2. 人別月次CSV
指定した職員・指定した年月について、1か月分の出退勤状況をCSV出力する。

### 入力条件
- 職員を1名選択
- 対象年月を選択

### 出力項目
| 項目 | 内容 |
|---|---|
| 日付 | 対象日 |
| 曜日 | 曜日 |
| 氏名 | 職員名 |
| 出勤 | その日の最初の出勤時刻 |
| 退勤 | その日の最後の退勤時刻 |
| 状態 | 未出勤 / 勤務中 / 退勤済 / 要確認 |

### 出力イメージ
| 日付 | 曜日 | 氏名 | 出勤 | 退勤 | 状態 |
|---|---|---|---|---|---|
| 2026/03/01 | 日 | 田中 | 08:30 | 17:15 | 退勤済 |
| 2026/03/02 | 月 | 田中 | 09:00 |  | 勤務中 |
| 2026/03/03 | 火 | 田中 |  |  | 未出勤 |

# 11. 管理者認証機能

## 11-1. 目的
編集機能を一般利用者から隠し、管理者のみ利用可能にする。

## 11-2. 認証方法
- 画面上の目立たない位置に「管理者」入口を設ける
- 管理者がパスワードを入力する
- 正しい場合のみ管理者モードを有効にする

## 11-3. 初版の運用
初版では、**代表のみが利用できる管理者機能**とする。

## 11-4. 管理者モード時の動作
- 一覧画面に「編集」列を表示
- 各職員の出勤・退勤時刻を修正可能
- 修正値を保存可能
- 一覧表示へ即時反映
- 必要に応じて管理者モード解除可能

---

# 12. 管理者編集機能

## 12-1. 目的
打刻漏れ、直帰、打刻忘れなどが発生した場合に、  
**管理者のみ** 出勤時刻・退勤時刻を修正できるようにする。

## 12-2. 編集条件
- 管理者パスワードを入力した利用者のみ編集可能
- 一般利用者には編集機能を表示しない

## 12-3. 編集対象
管理者は、指定した日付・職員について以下を編集できる。

- 出勤時刻
- 退勤時刻

## 12-4. 編集可能な内容
- すでに入力されている時刻の修正
- 空白の時刻への入力
- 必要に応じて時刻の空白化

## 12-5. 元ログの扱い
**修正前の元の打刻時刻は attendance_log に残す。**  
管理者修正では元ログを変更・削除しない。

## 12-6. 修正後の反映
一覧表示およびCSV出力では、修正データがある場合はそちらを優先して表示する。

---

# 13. エラー処理

| CSV対象なし | 「出力できるデータがありません」 |
| 管理者認証失敗 | 「パスワードが違います」 |

# 14. UI方針

- タブレット操作を前提とする
- ボタンは大きくする
- 文字サイズは大きくする
- 1〜2タップで打刻できる設計とする
- 一覧画面は見やすい表形式とする
- CSV出力ボタンは目立ちすぎない見た目とする
- 管理者入口は通常利用者の邪魔にならない位置に配置する
